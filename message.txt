# WebSocket (Socket.IO) Integration Guide — Vellox

## 1. Pendahuluan & Prasyarat

Backend Vellox menggunakan **Socket.IO**. Di sisi Frontend, gunakan **`socket.io-client` versi 4.x** agar kompatibel dengan server.

Instalasi:

```bash
npm install socket.io-client
```

## 2. Alur Koneksi & Autentikasi

### Endpoint URL

- Local backend (default): `http://localhost:3000`
- Socket.IO menggunakan path default: `/socket.io`

Catatan:

- Global prefix REST API (misalnya `/api`) **tidak** dipakai untuk WebSocket. Jadi URL koneksi Socket.IO tetap ke base host/port.

### Autentikasi (WAJIB)

Koneksi WebSocket **WAJIB** menyertakan JWT Access Token pada object `auth` saat inisialisasi client.

Backend akan membaca token dari:

- `socket.handshake.auth.token` (**yang wajib kamu set**)
- (fallback) header `Authorization: Bearer <token>`

Contoh koneksi (TypeScript):

```ts
import { io, type Socket } from 'socket.io-client';

const BACKEND_URL = 'http://localhost:3000';

export function createSocket(accessToken: string): Socket {
  return io(BACKEND_URL, {
    auth: {
      token: accessToken,
    },
  });
}
```

Perilaku server saat connect:

- Jika token **valid**, server akan menerima koneksi.
- Jika token **missing/invalid/expired**, server akan **memutus koneksi** (client akan menerima event `disconnect` / `connect_error` tergantung kondisi di sisi Socket.IO).

## 3. Mekanisme Room (Tenant)

Setelah event `connect` berhasil, frontend **WAJIB** melakukan emit event **`joinTenant`** agar socket bergabung ke “room” tenant.

### Event yang harus dikirim

- **Event**: `joinTenant`
- **Payload**:

```json
{ "tenantId": "uuid-string" }
```

### Konsekuensi jika tidak join room

Tanpa memanggil `joinTenant`, socket **tidak akan menerima broadcast event apapun** dari backend untuk tenant tersebut, karena backend hanya melakukan emit ke room tenant.

### Validasi server

Saat `joinTenant`:

- `tenantId` harus UUID v4.
- user harus benar-benar member tenant tersebut.

Jika gagal, server akan mengembalikan error (WsException) seperti:

- `tenantId is invalid`
- `Not a tenant member`
- `Unauthorized socket`

### Contoh pemanggilan `joinTenant` (disarankan pakai ACK)

`EventsGateway.joinTenant` mengembalikan response object, yang bisa kamu terima via acknowledgement callback Socket.IO.

```ts
socket.on('connect', () => {
  socket.emit('joinTenant', { tenantId }, (resp: unknown) => {
    // resp biasanya: { ok: true, tenantId: "..." }
    console.log('joinTenant resp', resp);
  });
});
```

## 4. Daftar Event (Listener)

Semua event di bawah ini dibroadcast ke room tenant. Artinya:

- Kamu **harus** sudah `joinTenant` untuk `tenantId` terkait.
- Payload akan selalu memiliki minimal `tenantId` dan biasanya ditambahkan `eventName` oleh backend.

### Event: Role

| Event name     | Kapan terjadi          | Ekspektasi payload (minimal)              |
| -------------- | ---------------------- | ----------------------------------------- |
| `role.created` | Role berhasil dibuat   | `{ eventName, tenantId, roleId, userId }` |
| `role.updated` | Role berhasil diupdate | `{ eventName, tenantId, roleId, userId }` |
| `role.deleted` | Role berhasil dihapus  | `{ eventName, tenantId, roleId, userId }` |

### Event: Tenant Member

| Event name                   | Kapan terjadi                           | Ekspektasi payload (minimal)                         |
| ---------------------------- | --------------------------------------- | ---------------------------------------------------- |
| `tenant.member.added`        | Member ditambahkan ke tenant            | `{ eventName, tenantId, userId, roleId, invitedBy }` |
| `tenant.member.removed`      | Member dikeluarkan / keluar dari tenant | `{ eventName, tenantId, userId, removedBy }`         |
| `tenant.member.role_updated` | Role member di tenant berubah           | `{ eventName, tenantId, userId, roleId, updatedBy }` |

### Event: Tenant Invite

| Event name                | Kapan terjadi     | Ekspektasi payload (minimal)                                  |
| ------------------------- | ----------------- | ------------------------------------------------------------- |
| `tenant.invite.created`   | Invite dibuat     | `{ eventName, tenantId, inviteId, email, roleId, invitedBy }` |
| `tenant.invite.cancelled` | Invite dibatalkan | `{ eventName, tenantId, inviteId, email, cancelledBy }`       |

### Contoh listen event dengan `socket.on()`

```ts
socket.on('role.created', (payload) => {
  // payload: { eventName: 'role.created', tenantId, roleId, userId, ... }
  console.log('role.created', payload);
});

socket.on('tenant.member.role_updated', (payload) => {
  console.log('tenant.member.role_updated', payload);
});

socket.on('tenant.invite.cancelled', (payload) => {
  console.log('tenant.invite.cancelled', payload);
});
```

## 5. Error Handling

Frontend perlu meng-handle error fungsional berikut:

- **`connect_error`**
  - Terjadi saat handshake gagal (misalnya konfigurasi client salah / server tidak bisa diakses).
  - Pada kasus autentikasi, dapat muncul jika token tidak diterima pada fase koneksi (tergantung behavior Socket.IO client).

- **`disconnect`**
  - Terjadi saat server memutus koneksi.
  - Pada backend Vellox, jika token missing/invalid/expired saat `handleConnection`, server akan memutus koneksi segera.

Contoh handler:

```ts
socket.on('connect_error', (err) => {
  console.error('socket connect_error', err);
});

socket.on('disconnect', (reason) => {
  // contoh reason: 'io server disconnect', 'transport close', dll.
  console.warn('socket disconnected', reason);
});
```
